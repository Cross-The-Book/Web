# Web
Web Programming Team Project

## Label
* feat: 새로운 기능 추가(개발)
* fix: 버그 수정
* refactor: 코드 리펙소링
* comment: 주석을 추가하거나 변경한 경우
* test: 테스트코드 추가, 변경 등을 한 경우
* rename: 파일 또는 폴더명을 수정하거나 옮기는 경우
* remove: 파일을 삭제하는 작업만 했을 경우
* docs: 문서 수정

## Branch Name
"[Label] 요약 내용" 형식으로 작성    
ex) [feat] main 화면 구현

## Branch Merge
만든 브랜치는 꼭!! develop 브랜치에 머지
